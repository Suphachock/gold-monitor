# Gold Price Monitor (ฮั่วเซ่งเฮง) → Telegram บน GitHub Actions

เฝ้าราคาทองฮั่วเซ่งเฮง แล้วส่งแจ้งเตือนเข้า Telegram เมื่อถึงเป้าซื้อ/ขาย
รันฟรี 24 ชม. บน **GitHub Actions** (เรียกทุก 5 นาทีอัตโนมัติ)

---

## วิธีติดตั้ง (ทำครั้งเดียว)

### 1) สร้าง Telegram Bot + หา CHAT_ID
1. เปิด Telegram คุยกับ **@BotFather** → พิมพ์ `/newbot` → ตั้งชื่อ → จะได้ **TOKEN**
   (หน้าตาแบบ `123456789:AAExxxxxxxxxxxxxxxxxxxxx`)
2. ทักบอทของคุณสัก 1 ข้อความ (เช่นพิมพ์ `hi`)
3. เปิดลิงก์นี้ในเบราว์เซอร์ (แทน `<TOKEN>` ด้วย token จริง):
   `https://api.telegram.org/bot<TOKEN>/getUpdates`
4. มองหา `"chat":{"id": 123456789 ...}` → เลข `id` นั่นคือ **CHAT_ID** ของคุณ

### 2) เอาโค้ดขึ้น GitHub
1. สร้าง repo ใหม่บน GitHub (จะ Private ก็ได้)
2. อัปโหลดไฟล์ทั้งหมดในโฟลเดอร์นี้ขึ้นไป (main.py, requirements.txt, state.json, .github/…)
   - ถ้ายังไม่คุ้น git ใช้ปุ่ม **Add file → Upload files** บนหน้าเว็บ GitHub ได้เลย

### 3) ใส่ TOKEN / CHAT_ID เป็น Secrets (สำคัญ อย่าใส่ในโค้ด!)
ที่ repo → **Settings → Secrets and variables → Actions → ปุ่ม New repository secret**
เพิ่ม 2 อัน:

| Name | Secret |
|------|--------|
| `TELEGRAM_TOKEN` | token จากข้อ 1 |
| `TELEGRAM_CHAT_ID` | chat id จากข้อ 1 |

### 4) (ไม่บังคับ) ปรับความไว "ขยับแรง"
แท็บเดียวกัน เลือก **Variables → New repository variable** เพิ่มได้:

| Name | ตัวอย่างค่า | ความหมาย |
|------|-----------|----------|
| `MOVE_THRESHOLD` | `100` | เตือนเมื่อราคาขยับจากราคาอ้างอิงเกิน **กี่บาท** (0 = ปิด) |
| `MOVE_PERCENT` | `0.3` | หรือเตือนเมื่อขยับเกิน **กี่ %** (0 = ปิด) |
| `GOLD_TYPE` | `HSH` | ชนิดทอง: HSH / REF / JEWEL |

- ใช้เกณฑ์ไหนถึงก่อนก็เตือน (จะตั้งทั้งบาทและ % พร้อมกันก็ได้)
- ถ้าไม่ตั้งอะไรเลย จะใช้ค่า default = ขยับ **≥ 100 บาท** ถึงเตือน
- **เตือนบ่อยเกิน?** เพิ่ม `MOVE_THRESHOLD` เป็น 200–300 · **เตือนน้อยไป?** ลดลง

### 5) ทดสอบ
ที่ repo → แท็บ **Actions** → เลือก workflow **Gold Price Monitor** → ปุ่ม **Run workflow**
ดู log ว่าดึงราคาได้ไหม ถ้าราคาถึงเป้าจะมีข้อความเข้า Telegram ทันที

เสร็จแล้ว! หลังจากนี้มันจะรันเองทุก ~5 นาที ตลอด 24 ชม.

## เรื่องที่ควรรู้

- **วิธีตรวจจับการขยับ:** ระบบเก็บ "ราคาอ้างอิง" (anchor) ไว้ในไฟล์ `state.json`
  พอราคาขยับจากจุดนั้นเกินเกณฑ์ จะเตือน 1 ครั้งแล้วเลื่อนอ้างอิงมาที่ราคาปัจจุบัน
  จึงไม่สแปม และวัดการขยับ "รอบต่อไป" จากจุดใหม่เสมอ
- **รอบแรก** ที่รันจะยังไม่เตือน (แค่ตั้งราคาอ้างอิงเริ่มต้น) รอบถัดไปถึงเริ่มเทียบ
- **cron อาจดีเลย์:** ช่วงที่ GitHub คนใช้เยอะ ตารางเวลาอาจคลาดไปบ้าง (บางทีช้า 5–15 นาที) เป็นเรื่องปกติของ tier ฟรี
- **โควตาฟรี:** บัญชีฟรีมี 2,000 นาที/เดือน แต่ละรอบนี้ใช้ไม่ถึง 1 นาที → เหลือเฟือ
- **repo Private จะกินโควตา** ส่วน repo **Public รันฟรีไม่จำกัด** (แต่โค้ดจะเปิดเผย — token ปลอดภัยเพราะอยู่ใน Secrets ไม่ใช่ในโค้ด)
- ถ้า repo เงียบ (ไม่มี push) เกิน 60 วัน GitHub จะพักตาราง cron ให้เอง — แค่เข้าไปกด Run workflow หรือ push อะไรสักอย่างก็กลับมาทำงาน

## รันในเครื่องตัวเอง (เทสก่อน)
```bash
pip install -r requirements.txt
# Windows PowerShell
$env:TELEGRAM_TOKEN="ใส่ token"; $env:TELEGRAM_CHAT_ID="ใส่ chat id"; python main.py
```
